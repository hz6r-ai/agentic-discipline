# قائمة فحص ما قبل التسليم (Pre-Delivery Checklist)

قبل تسليم أي مخرج تقني نهائي للمستخدم، تأكد:

- [ ] السبب الجذري مكتوب صراحة مع دليله، مو "حدس عام"
- [ ] كل ادعاء تقني موسوم [مؤكد] أو [افتراض] — ولا فيه افتراض غير محلول يُبنى عليه القرار النهائي
- [ ] نطاق الأثر (Blast Radius) قُدِّر صراحة قبل أي تعديل واسع
- [ ] التحقق المنطقي تم (التعديل يطابق التشخيص فعلًا)
- [ ] التحقق التشغيلي تم بدليل فعلي حقيقي (لوق/اختبار/وصف دقيق) — مو افتراض أن الكود "لازم يشتغل"
- [ ] بصمة الثقة للمكوّنات الحرجة ≥ 80%، وإلا مذكور صراحة أنها أقل والسبب
- [ ] دفتر الحالة محدَّث بكل الأحداث الجديدة (DECISION/ACTION/VERIFY) قبل إنهاء الرد
- [ ] لو فيه محاولة سابقة فشلت في نفس المهمة: الحل الحالي مختلف جذريًا، مو نفس الفكرة بصياغة جديدة
- [ ] لا توجد أي صيغة اختصار توحي بالاكتمال ("..."، "(يستمر)"، إلخ) في مخرج يُفترض أنه كامل
- [ ] لو المخرج طويل جدًا فعليًا ويتجاوز حدود رد واحد: مقسّم لأجزاء كاملة كل واحد منها، مو ملف وحيد فيه فجوات
